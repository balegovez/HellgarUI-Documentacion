Spacing

Margen, padding y direcciones · sistema responsivo de 6 escalones.

En esta página
  • La escala
  • Sintaxis
  • Padding

La escala

El sistema se basa en seis pasos calculados a partir del $spacer base (1rem = 16px). Sirve igual para padding y para margin.

Valores
Token Multiplicador rem px
0 0 0 0px
1 0.25 0.25rem 4px
2 0.5 0.5rem 8px
3 1 1rem 16px
4 1.5 1.5rem 24px
5 3 3rem 48px

Sintaxis

El patrón es {propiedad}{lado}-{tamaño} y opcionalmente {propiedad}{lado}-{breakpoint}-{tamaño} para responsivo.

Atajos
  • m = margin · p = padding
  • t = top · b = bottom · s = start (left) · e = end (right)
  • x = horizontal (start + end) · y = vertical (top + bottom)
  • Sin sufijo de lado: aplica a los cuatro (ej. p-3, m-4). 
<div class="p-3">padding 1rem en los 4 lados</div>
<div class="mt-4">margin-top 1.5rem</div>
<div class="px-5 py-2">padding horizontal 3rem y vertical 0.5rem</div>
<div class="mb-md-4">margin-bottom 1.5rem desde el breakpoint md</div>

Padding

.p-0 hasta .p-5 sobre los cuatro lados. Las cajas de adentro muestran cuánto crece el padding.

Ejes horizontal y vertical

Para evitar tener que escribir dos clases, usá x (izquierda + derecha) y y (arriba + abajo).

Vista previa
.px-4 .py-2
.px-2 .py-4 
<div class="px-4 py-2">Ancho con padding y poco vertical</div>
<div class="mx-3 my-5">Márgenes en los dos ejes</div>

Responsivo

Cualquier clase de spacing acepta un breakpoint en el medio: sm, md, lg, xl, xxl. La regla aplica desde ese ancho hacia arriba (mobile-first).

Idea

Si un bloque debe tener poco padding en mobile y bastante en desktop, lo escribís así: 

<div class="p-2 p-md-4 p-lg-5">
    Padding crece a medida que la pantalla crece.
</div>
